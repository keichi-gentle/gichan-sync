1. AI 사용 환경 및 현황
- 회사 복귀 후 약 2주정도 사용(3월 중순부터)
- 현재 PC의 AI 구동 환경이 안 좋아서(AI 2개 돌리면 회사 그룹웨어 사용도 힘듬) 최소한 SSD 용량 및 속도 개선을 위해 SSD 구매요청 (3/31 도착)
- 3/19 팀 AI 활용 회의(하네스 개념 설명) => 관련 SKILLS/PLUG-IN 설치후 3/23부터 약 2주간사용

2. 현업 프로젝트 적용
- 현업은 아직 설계 단계만 적용하여 사용 (서울스마트관제, 대전1호선)
- 요구사항 작업 1차 (수집 & 분석) -> AI로 요구사항 작업 2차(분석/검토) -> AI와 설계 작업 동시 진행
- 기존에 설계한 부분도 AI로 설계 검증 및 설계 자료 수정 보완에 활용 -> 설계 회의

3. 프로젝트 전반에 걸친 활용 시험
- 개인 테스트 프로젝트로 하네스 구축/사용 TEST
■ 1단계: 프로젝트 컨텍스트 파악
■ 2단계: 요구사항문서 초안 작성
■ 3단계: 추가 논의 및 반영 (시나리오/데이터/GUI 레이아웃)
■ 4단계: 아키텍처 방향 결정 (데이터/기능구현/UI/캐쉬/비동기처리)
■ 5단계: PRD 작성 (프로젝트 구조 및 데이터 모델 확정)
■ 6단계: UI 목업 HTML 작성 (COLOR PALETTE/THEME/시안/UX개선)
■ 7단계: UI 폴리싱(UI Polishing)
■ 8단계: Deploy, 산출물 및 기록물 정리
* 프로젝트: 실제 45단계, 설계~배포까지 10일 정도 소요, 각 단계별 AI로 git처리, 파생 프로젝트 진행중

4. 각 공정별 AI 활용 경험 의견
- 요구사항 정리/분석이 빠르게 진행됨
- 제3자(AI)의 설계 검토를 빠르게 받을 수 있음(회의 잡고 모여서 토론하는 시간대비 엄청난 효율)
 * 사람들과 설계 회의 하기 전 더 많은 검토가 이루어진 기초 설계가 준비됨.
- GUI 'MOCK-UP'이 빠르게 생성되서 UX 검토가 빠르게 진행됨. 현재 가장 크게 체감되는 부분
- 프로그램 ProtoType이 빠르게 제작되어 프로그램의 사전 시험이 빠르게 진행될 수 있음
- 구현 기술 검토 및 파생 테스트가 빠르게 진행됨

5. 기타 의견
- 반복되는 수정 작업 발생으로 효율이 매우 떨어지는 현상이 있음 => 관련 md나 prompt를 잘 작성하여 문제점 보완
- 초기 각종 AI 활용 개념을 접하고, 여러가지 도구들을 설치하며, 다양한 요소들의 각 사용법 등을 적응하는 시기로 부분적으로 시간이 더 지연되는 현상이 발생
- 초기 적응 시기를 줄이고 체계적으로 새로운 기술을 익히기 위해 AI 사용 패턴 등을 분석해보기 위해 AI 주요 협업 과정과 내용을 AI를 이용해 자동으로 기록하고, 추후 AI를 이용해 분석해 볼 예정
