# 기능 설계 — 예방접종 일정/기록 관리

작성일: 2026-04-20
대상 버전: 미정 (v3.2.x 또는 v3.3.0 유력)
상태: **초안 검토 중** — B안 또는 C안 방향으로 심화 설계 예정

---

## 1. 배경/목적

아기 부모가 국가 권장 예방접종 일정과 실제 접종 이력을 한눈에 파악하고 관리할 수 있는 기능을 추가한다.

**주요 목적**:
- 아기 생년월일 기준으로 **권장 접종 시기를 자동 계산**하여 표시
- 접종 완료 여부(접종일, 병원명)를 체크하여 놓침 방지
- 다가오는 접종, 지연된 접종 가시성 확보

**기존 인프라 활용 가능 요소**:
- `babyBirthDate` 설정 이미 존재 → 개월 수 기반 일정 계산 가능
- "건강관리" 이벤트 카테고리 기존 존재 → 접종 기록 저장처로 활용 가능
- Firestore 설정 공유 구조 (`users/{uid}/settings/app`) → 스케줄 마스터 저장처로 활용 가능

---

## 2. 기능 개요

**제공 정보**:
- 예방접종 권장 스케줄 (백신명, 차수, 권장 개월/주차, 마감 개월)
- 아기 생년월일 기준 각 접종의 권장 시점/마감일 자동 계산
- 접종 완료 여부 + 접종일 + 병원명 체크
- 상태 분류: 완료 / 임박(권장일 ±2주) / 지연(마감 초과 미접종) / 예정(미래)

---

## 3. 구현 옵션 비교

### A안. 자동 내장 (국가 표준 스케줄 하드코딩)

- 한국 국가예방접종(NIP) 14종 스케줄을 앱 코드(JSON/상수)에 내장
- 별도 설정 없이 바로 사용

| 장점 | 단점 |
|------|------|
| 설치/설정 0단계 | 국가 규정 개정 시 앱 업데이트 배포 필요 |
| 신뢰할 수 있는 기본값 제공 | 병원별/가정별 일정 조정 불가 |
| 구현 단순 (JSON 한 벌) | 국외 사용자 대응 불가 (현재 한국 전용이라 OK) |

### B안. 관리자 수동 설정

- 관리자가 설정 탭에서 백신 목록을 직접 등록 (이름, 권장 개월, 차수, 마감 개월)
- Firestore `users/{adminUid}/settings/app` 에 저장 → 사용자도 공유 열람

| 장점 | 단점 |
|------|------|
| 완전한 유연성 | 관리자가 최초 14종을 일일이 입력해야 함 |
| 병원별 맞춤 스케줄 가능 | 실수/누락 위험 |
| 앱 업데이트 없이 변경 가능 | 관리 UI 복잡도 증가 |

### C안. 하이브리드 (기본값 + 커스터마이징)

- 앱에 기본 국가 스케줄 내장 (A안 방식)
- 설정 탭에서 항목별 "편집/추가/삭제" 가능 (B안 방식)
- 리셋 버튼으로 기본값 복원 가능

| 장점 | 단점 |
|------|------|
| 즉시 사용 + 유연성 모두 확보 | UI가 가장 복잡 |
| 개정 시 앱 업데이트 + 기존 사용자 커스텀 존중 | 구현 공수 A안보다 큼 |

---

## 4. 권장안 및 방향성

**현재 검토 방향: B안 또는 C안**
- 초기 검토에서 C안이 "즉시 사용 + 유연성"을 모두 제공하여 우세로 제시됨
- 단, B안도 배제하지 않고 비교 심화 예정 (관리자 부담 vs 앱 업데이트 종속성 트레이드오프 재검토 필요)

**A안은 단독으로는 채택하지 않음** (개정 시 앱 배포 종속성이 강해 장기적으로 불리)

## 5. 데이터 구조 (C안 기준 초안)

### 5-1. 스케줄 마스터 (공유 설정)

Firestore `users/{adminUid}/settings/app` 하위 또는 별도 문서:

```json
{
  "vaccinationSchedule": [
    { "id": "bcg-1",     "name": "BCG",       "doseNumber": 1, "recommendedMonths": 0, "deadlineMonths": 4 },
    { "id": "hepb-1",    "name": "B형간염",   "doseNumber": 1, "recommendedMonths": 0, "deadlineMonths": 1 },
    { "id": "hepb-2",    "name": "B형간염",   "doseNumber": 2, "recommendedMonths": 1, "deadlineMonths": 3 },
    { "id": "hepb-3",    "name": "B형간염",   "doseNumber": 3, "recommendedMonths": 6, "deadlineMonths": 12 },
    { "id": "dtap-1",    "name": "DTaP",      "doseNumber": 1, "recommendedMonths": 2, "deadlineMonths": 4 }
    // ... (한국 NIP 기준 14종, 총 ~30+ 항목)
  ]
}
```

### 5-2. 접종 이력

기존 `events` 컬렉션의 '건강관리' 카테고리 레코드에 필드 추가:

```json
{
  "category": "건강관리",
  "subType": "vaccination",
  "vaccineId": "dtap-1",
  "vaccineName": "DTaP",
  "doseNumber": 1,
  "date": "2026-06-15",
  "hospital": "○○소아과",
  "note": ""
}
```

**선택지**: "건강관리" 하위 subType 방식 vs 별도 `vaccinations` 컬렉션 분리 → 심화 검토 시 결정

---

## 6. 상태 분류 로직

아기 생년월일 기준 현재 개월 수 계산 → 각 스케줄 항목과 매칭:

| 상태 | 조건 |
|------|------|
| **완료** | 해당 `vaccineId + doseNumber` 로 접종 기록 존재 |
| **임박** | 접종 기록 없음 + 현재 시점이 권장일 ±2주 이내 |
| **지연** | 접종 기록 없음 + 마감 개월 초과 |
| **예정** | 접종 기록 없음 + 권장일 아직 도래 전 |
| **해당없음** | 개월 수 맞지만 삭제된/면제된 백신 (커스텀 삭제) |

## 7. UI 구성

| 위치 | 내용 | 권한 |
|------|------|------|
| **설정 탭 > 예방접종 일정 관리** | 스케줄 마스터 편집 (백신 추가/수정/삭제, 기본값 리셋) | 관리자만 |
| **신규 탭 "예방접종" 또는 현황탭 카드** | 아기 현재 개월 기준 다가오는/지연된/완료 백신 리스트, 권장일 vs 실제 접종일 | 모두 |
| **기록 탭 > 건강관리** | "예방접종" 하위 옵션 → 백신 선택 드롭다운 + 차수 + 병원 | 사용자+관리자 |
| **리포트 탭 (선택)** | 접종 완료율, 월별 접종 타임라인 | 모두 |

**신규 탭 vs 현황탭 카드** 선택: 심화 검토 시 결정 (데이터량/중요도에 따라)

## 8. 단계별 구현 제안 (MVP 접근)

전체 구현이 부담되면 단계별로:

### 1단계 (소) — 기본 가시성
- 설정 탭에 예방접종 스케줄 편집 UI
- 현황탭에 "다음 접종" 카드 1개 (임박 항목만)

### 2단계 (중) — 기록 연동
- 기록 탭 건강관리에서 백신명 선택 가능
- 접종 실제일 기반으로 완료 상태 표시

### 3단계 (중) — 전용 화면
- 전용 "예방접종" 탭
- 리스트/상세, 완료율, 지연 알림

## 10. 추가 고려사항

1. **백신명 표준화**: 국가 명칭(예: "DTaP") vs 일반 명칭(예: "디프테리아") 표기 통일
2. **차수 표기**: "1차/2차/3차" vs 국제 표기 "Dose 1/2/3"
3. **접종 기록 일원화**: 기존 건강관리 카테고리에 녹일지, 별도 `vaccinations` 컬렉션으로 분리할지
4. **알림/뱃지**: 임박/지연 상태를 전광판 또는 현황탭에 뱃지로 강조 표시 (향후 확장)
5. **국가 규정 개정 대응**: C안 채택 시에도 기본 스케줄이 앱 코드에 있으므로 개정 시 업데이트 배포 필요 — 대응 정책 수립
6. **복수 아기 대응**: 현재 앱은 단일 아기 전제 → 예방접종도 단일 아기 기준으로 설계, 복수 아기 지원은 별도 논의
