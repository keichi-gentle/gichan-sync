■ 프로젝트 배경
- 기존 LifeRecord(WPF)는 로컬 Excel만 사용, ExtendPrj(PWA)는 수동 Import
- PC에서 기록하면 폰에서 즉시 조회 가능한 실시간 동기화 필요
- Firebase Firestore 채택 (무료, 실시간 동기화, 오프라인 지원)

■ 기술 검토 (선택지 비교)
- Firebase: 실시간 O, .NET/JS SDK O, 무료 범위 충분, 난이도 낮음 → 채택
- Supabase: 실시간 O, REST API, 무료 범위 충분, 난이도 중간
- Google Sheets: 실시간 X (폴링만), API 제한, 난이도 높음
- JSON on Google Drive: 실시간 X, 충돌 위험, 난이도 높음

■ 프로젝트 구조 결정
- B방식 채택: 기존 프로젝트 소스를 복사하여 GichanSync 내에서 독립 작업
- src/wpf/ — LifeRecord 복사본 + Firebase 기능 추가
- src/pwa/ — ExtendPrj/gichan-viewer 복사본 + Firebase 연동
- src/migration/ — Excel→Firestore 마이그레이션 콘솔 앱
